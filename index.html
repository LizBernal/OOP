Los 4 elementos de Object Oriented Programming (OOP)

¿Qué es?

La Programación Orientada a Objetos (en inglés, Object Oriented Programming o OOP), es un paradigma de programación que organiza el código en unidades llamadas objetos. Estos objetos representan entidades del mundo real (como personas, coches, etc.) y tienen atributos (características) y métodos (acciones).

OOP se sustenta en cuatro elementos o pilares fundamentales:

Abstracción

La abstracción consiste en identificar las características esenciales de un objeto y omitir los detalles irrelevantes. Es como crear un modelo simplificado de un objeto del mundo real.

  • Ejemplo:

    • Al modelar un coche, la abstracción nos permite enfocarnos en atributos como marca, modelo y color, sin preocuparnos por los detalles mecánicos internos.

    Encapsulación

    La encapsulación es el mecanismo que oculta la implementación interna de un objeto, exponiendo solo una interfaz para interactuar con él. Esto protege los datos internos del objeto y evita modificaciones no deseadas.

  • Ejemplo:

    • Un televisor es un buen ejemplo. Tú interactúas con él a través de botones y un control remoto (la interfaz), sin necesidad de saber cómo funciona el circuito interno.

    Herencia

    La herencia permite crear nuevas clases (objetos) basadas en clases existentes, heredando sus atributos y métodos. Es como crear una jerarquía de clases, donde las clases hijas heredan las características de las clases padres.

  • Ejemplo:

    • Un perro y un gato son animales, por lo que pueden heredar atributos como el número de patas y métodos como comer o dormir.

    Polimorfismo

    El polimorfismo permite que diferentes clases implementen métodos con el mismo nombre de diferentes maneras. Lo cual significa que objetos de diferentes clases pueden responder de manera diferente al mismo mensaje, es como tener objetos que pueden adoptar diferentes formas según el contexto.

  • Ejemplo:

    • Un pájaro y un avión pueden volar, pero lo hacen de manera diferente. El polimorfismo permite tratar a ambos objetos como si fueran de un tipo más general (en este caso, "volador"), sin tener que preocuparnos por sus diferencias internas.
